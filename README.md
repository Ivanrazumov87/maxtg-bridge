# Двусторонний мост MAX ⇄ Telegram

Бот зеркалит сообщения между мессенджером **MAX** и **Telegram** в обе стороны.
Каждый чат MAX отображается **отдельной темой (топиком)** в форум-супергруппе
Telegram: новые сообщения из MAX попадают в свою тему, а написанное вами в тему
уходит обратно в соответствующий чат MAX.

> ⚠️ **Дисклеймер.** Бот использует токен веб-сессии вашего аккаунта MAX (не
> официальный bot-API). Это неофициальный способ, и **есть ненулевой риск
> блокировки аккаунта MAX** за автоматизацию. Используйте на свой страх и риск.

---

## ✨ Возможности

- 🔄 **Двусторонняя** пересылка: MAX → Telegram и Telegram → MAX
- 🧵 Каждый чат MAX = отдельная **тема** в одной супергруппе Telegram
- 🌐 По умолчанию пересылаются **все** чаты MAX (можно сузить whitelist'ом)
- 🆕 Темы создаются автоматически при первом сообщении из нового чата
- ♻️ Если тему удалить в Telegram — мост **пересоздаст** её при следующем сообщении
- 📸 Медиа: текст, фото, видео и файлы
- 🛡 Защита от эхо-петли (ваш ответ из Telegram не дублируется обратно)
- 🔌 Автоматическое переподключение при обрыве связи

### Что поддержано по направлениям

| Формат | MAX → Telegram | Telegram → MAX |
|---|:---:|:---:|
| Текст | ✅ | ✅ |
| Фото | ✅ | ✅ |
| Видео | ✅ | ✅ (до 20 МБ¹) |
| Файлы | как «необраб. файл» | ✅ (до 20 МБ¹) |

¹ Telegram-боты не могут скачивать файлы больше 20 МБ — на крупные файлы мост
пришлёт уведомление в тему.

## 🔑 Пошаговая настройка

### Шаг 1. Скопируйте файл конфигурации

```bash
cp .env.example .env
```
Дальше по шагам заполняете значения в `.env`.

### Шаг 2. Получите токен MAX

Токен — это ключ вашей веб-сессии MAX.

> 💡 **Важно: используйте ОТДЕЛЬНУЮ сессию.** Если бот будет работать на том же
> токене, что и ваш обычный вход в `web.max.ru`, MAX будет воспринимать это как
> «одна сессия в двух местах» и периодически рвать соединение бота. Поэтому
> получите токен из **окна инкогнито** — это создаст независимую сессию.

1. Откройте **новое окно в режиме инкогнито**:
   - Chrome/Edge: `Ctrl + Shift + N`
   - Firefox: `Ctrl + Shift + P`
2. Зайдите на **https://web.max.ru** и войдите в аккаунт (по QR или номеру).
3. Нажмите **F12** → вкладка **Application** (или «Хранилище» / Storage).
4. Слева: **Local Storage** → `https://web.max.ru`.
5. Найдите ключ с токеном (обычно содержит `auth` / `token`, напр.
   `__oneme_auth_token`). Скопируйте **значение** (длинная строка).
6. Вставьте его в `.env` как `MAX_TOKEN=...`

> Можно не закрывать окно инкогнито, чтобы сессия жила. Если токен «протухнет»
> (бот не сможет залогиниться) — повторите шаг и обновите `MAX_TOKEN`.

### Шаг 3. Создайте Telegram-бота

1. Напишите в Telegram боту **[@BotFather](https://t.me/BotFather)**.
2. Команда `/newbot` → придумайте имя и username.
3. Скопируйте выданный токен → вставьте в `.env` как `TG_BOT_TOKEN=...`

### Шаг 4. Создайте форум-супергруппу и узнайте её ID

Все чаты MAX будут темами **одной** супергруппы Telegram.

1. Создайте **группу** в Telegram (добавьте любого второго участника — например,
   второй свой аккаунт; Telegram сам превратит её в супергруппу).
2. В **настройках группы** включите **«Темы» (Topics)**.
3. Добавьте в группу вашего бота из шага 3 и сделайте его **администратором**
   с правом **«Управление темами»** (Manage Topics).
4. Узнайте **ID группы**: добавьте в неё бота
   [@getmyid_bot](https://t.me/getmyid_bot) (или [@my_id_bot](https://t.me/my_id_bot)),
   он покажет ID группы вида **`-100...`**. После можно убрать вспомогательного бота.
5. Вставьте этот ID в `.env` как `TG_GROUP_ID=-100...`

### Шаг 5. (необязательно) Whitelist чатов

В `.env`:
- `MAX_CHAT_IDS=` пусто → пересылаются **все** чаты MAX.
- `MAX_CHAT_IDS=123,456` → только эти чаты. ID чата виден в адресной строке
  `web.max.ru/ID_ЧАТА`.

### Итоговый `.env`

```
MAX_TOKEN=ваш_длинный_токен_из_шага_2
MAX_CHAT_IDS=
TG_BOT_TOKEN=токен_из_шага_3
TG_GROUP_ID=-100xxxxxxxxxx
MONITOR_ID=
```

## 🚀 Запуск

```bash
python starter.py
```
`starter.py` запускает `main.py` и автоматически перезапускает его при падении.
Если в консоли видите `Имя: ... | ID: ...` и `Telegram poller запущен` — мост
работает.

### Проверка

- **MAX → Telegram:** напишите в любой чат MAX → в супергруппе появится тема с
  названием чата и сообщением.
- **Telegram → MAX:** напишите в эту тему → сообщение уйдёт в соответствующий
  чат MAX.

## 🖥 Запуск на сервере 24/7 (Ubuntu + systemd)

Чтобы мост работал постоянно (не завися от вашего компьютера), разместите его на
VPS. Пример для Ubuntu 22.04/24.04:

```bash
# 1. Установить зависимости системы
sudo apt update && sudo apt install -y python3-venv python3-pip git

# 2. Склонировать проект
sudo mkdir -p /opt/maxtg && cd /opt/maxtg
git clone https://github.com/<ВАШ_ЛОГИН>/<ИМЯ_РЕПО>.git .

# 3. Виртуальное окружение и зависимости
python3 -m venv venv
./venv/bin/pip install -r requirements.txt

# 4. Создать .env (как в шагах выше)
cp .env.example .env && nano .env
```

Создайте systemd-сервис `/etc/systemd/system/maxtg.service`:

```ini
[Unit]
Description=MAX <-> Telegram bridge
After=network-online.target
Wants=network-online.target

[Service]
Type=simple
WorkingDirectory=/opt/maxtg
ExecStart=/opt/maxtg/venv/bin/python -u main.py
Restart=always
RestartSec=5
Environment=PYTHONUNBUFFERED=1
Environment=PYTHONUTF8=1
StandardOutput=append:/var/log/maxtg.log
StandardError=append:/var/log/maxtg.log

[Install]
WantedBy=multi-user.target
```

Запуск и автозагрузка:

```bash
sudo systemctl daemon-reload
sudo systemctl enable maxtg     # автозапуск при загрузке сервера
sudo systemctl start maxtg
sudo systemctl status maxtg     # проверить статус
tail -f /var/log/maxtg.log      # смотреть логи
```

> На сервере проблемы «двух сессий» нет — бот единственный потребитель токена.

## 🧩 Как это устроено

- При первом сообщении из чата MAX бот создаёт тему (`createForumTopic`) и
  запоминает связь `chat_id ⇄ message_thread_id` в файле `bridge_state.json`.
- Связи и offset Telegram переживают перезапуск (хранятся на диске).
- Приём из Telegram — long-polling (`getUpdates`), вебхук не нужен.

Файлы:
- `main.py` — точка входа, обработчик сообщений MAX
- `bridge.py` — оркестратор MAX⇄Telegram, маршрутизация по темам
- `max.py` — клиент MAX (WebSocket-протокол)
- `telegram.py` — работа с Telegram Bot API
- `store.py` — хранилище связей чат↔тема (JSON)
- `classes.py`, `filters.py`, `errors.py` — модели и утилиты клиента MAX
- `get_token.py` — вспомогательный скрипт авторизации MAX по номеру (если
  доступно; основной способ — токен из браузера, см. Шаг 2)
- `starter.py` — запуск с авто-перезапуском

## 🆘 Troubleshooting

**Бот не логинится / `Логин MAX отклонён`**
Токен невалиден или протух. Получите новый (Шаг 2). Также MAX не любит большой
`chatsCount` — он зашит безопасным (100).

**Частые обрывы `no close frame received or sent`**
Обычно — конфликт сессий: тот же токен используется ещё где-то (открыт
`web.max.ru` в обычном браузере). Используйте токен из **отдельной/инкогнито**
сессии (Шаг 2). Мост сам переподключается, но отдельный токен убирает причину.

**Темы не создаются**
Бот должен быть **админом** супергруппы с правом **«Управление темами»**, а у
группы включены **«Темы»** (Шаг 4).

**Сообщения из Telegram не уходят в MAX**
Пишите именно **в тему**, привязанную к чату MAX (созданную мостом). Сообщения
в «General» или в темы, созданные вручную, не маршрутизируются.

**Видео из Telegram не отправляется**
Telegram-боты не качают файлы >20 МБ — это ограничение платформы. Мост уведомит
в теме. Маленькие видео отправляются.
